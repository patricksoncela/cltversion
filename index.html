Controle de Versão

O que é?

de versão se trata da prática de de rastrear e gerenciar as informações de um projeto. Para isso são utilizados os sistemas de controle de versão, eles permitem que você reverta para estado anterior determinados arquivos ou um projeto inteiro, compare as mudanças ao longo do tempo, etc.

Sistemas Locais de Controle de Versão

Muitas pessoas utilizam este método, copiando os arquivos de um projeto para outra pasta, porém apesar de ser muito fácil esse método também apresenta grande chance de apresentar problemas, como o usuário esquecer qual pasta havia deixado os arquivos e deleta-la. Para isso há sistemas especiais específicos como o RCS, que salva conjuntos de alterações em um lugar especial no disco.

Sistemas Centralizados de Controle de Versão(CVCSs) E Sistemas Descentralizados de Controle de Versão(DVCSs).

Ambos foram criados tendo em vista o trabalho em equipe e a colaboração com usuários de outras máquinas, porém, enquanto os CVCS tem um servidor centralizado que contém todos os arquivos de controle de versão, nos DVCS, os usuários podem duplicar e clonar localmente cada reposítorio de forma completa, tendo um backup não só remotamente como localmente.

Controle de Mudanças

O controle de mudanças se refe a prática de gerenciar e monitorar as alterações de um projeto, envolvendo o planejamento dessas alterações, em seguida, a implementação dessas mudanças de forma segura e controlada, com utilizando do controle de versão para isso. Após isso também é testado e garantido que as mudanças não introduzam novos problemas, além de registrar estas mudanças em uma documentação para facilitar o entendimento do projeto.

Integração Contínua

A integração contínua (CI), se trata de uma prática utilizada por equipes de desenvolvimento de software em que com frequência as alterações em um código são mescladas em um repositório central, o objetivo dessa prática é encontrar e corrigir os erros da forma mais rápida possível.
